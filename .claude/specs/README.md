# Specs

Feature specifications and requirements documents for the a2-school platform.

## Structure

```
specs/
  <feature-name>.md    # Spec for a specific feature or module
```

## Naming Convention

- Use kebab-case for filenames: `student-enrollment.md`, `fee-management.md`
- Each spec should cover: Overview, User Stories, Acceptance Criteria, and API contract (if applicable).
